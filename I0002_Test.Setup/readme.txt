Windows 服务的安装程序包.

1.首先，Windows 服务代码的编写
  注意设置服务的启动方式为 自动.

2.在 ProjectInstaller.cs 中，编写相关处理逻辑。
  也就是 安装完毕后，尝试启动 Windows 服务。
  卸载以前，先尝试停止 Windows 服务.

3.在安装项目的 文件系统视图里面，选择 “应用程序文件夹”， 鼠标右键。 选择 添加 --> 项目输出。
  确认对话框中，使用默认选择的 “主输出”，  按“确定”按钮。


4.选择 安装项目， 鼠标右键，选择 视图-->启动条件
  在 安装/提交/卸载 的地方，分别把  主输出来自... 加入进来。

  注：
在添加自定义操作时，一定要在Install中也添加上输出。
可能是因为在Install阶段进行.installstate文件的创建。
所以如果不在Install中添加输出的话，就会提示找不到相应的.installstate文件。
比如在自定义操作中重写了Commit方法，只想在安装结束后进行自定义操作，但是在制作安装文件添加自定义操作时，应该同时在Install和Commit中添加输出。


5.选择 安装项目. 在下面的属性的地方，InstallAllUsers 设置为 True

6.选择 安装项目， 鼠标右键，选择 视图-->用户界面
  在 安装文件夹 的位置， 把 InstallAllUsersVisible 设置为 False
